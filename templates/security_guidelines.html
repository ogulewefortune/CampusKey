üîê Section 1: Password Policy

1.1 Password Requirements

All CampusKey University accounts must comply with the following password requirements:

  • Minimum Length: 12 characters (16 characters recommended for administrative accounts)
  • Character Requirements: Must include at least one uppercase letter, one lowercase letter, one number, and one special character (!@#$%^&*()_+-=[]{}|;:,.<>?)
  • Complexity: Cannot contain your username, name, or common dictionary words
  • Uniqueness: Cannot reuse your last 5 passwords
  • Expiration: Passwords must be changed every 90 days (60 days for faculty/staff)

1.2 Password Prohibitions

The following practices are strictly prohibited:

  • Sharing passwords with anyone, including family members, friends, or colleagues
  • Writing passwords down on paper or storing them in unencrypted files
  • Using the same password for multiple accounts
  • Using personal information (birthdates, addresses, pet names) in passwords
  • Using common patterns or sequences (123456, password, qwerty)
  • Storing passwords in browser auto-fill without master password protection

1.3 Password Management

Recommended practices for managing your passwords:

  • Use a Password Manager: University-approved password managers include LastPass, 1Password, or Bitwarden
  • Two-Factor Authentication: Always enable MFA/2FA when available
  • Regular Audits: Review your account activity monthly
  • Breach Monitoring: Use Have I Been Pwned or similar services to check for compromised credentials

üîë Section 2: Multi-Factor Authentication (MFA)

2.1 MFA Requirements

All CampusKey University accounts are required to enable Multi-Factor Authentication:

  • Faculty & Staff: Mandatory within 30 days of account creation
  • Students: Mandatory for accessing grades, financial aid, and housing systems
  • Administrative Accounts: Mandatory with hardware security keys for privileged access

2.2 Approved MFA Methods

  • Authenticator Apps: Google Authenticator, Microsoft Authenticator, Authy
  • SMS/Email: Secondary verification method (less secure, not recommended)
  • Hardware Tokens: YubiKey, FIDO2 security keys (required for admin accounts)
  • Biometric Authentication: Fingerprint, Face ID (approved mobile devices only)

üì± Section 3: Device Security

3.1 University-Owned Devices

  • Must have device encryption enabled (BitLocker for Windows, FileVault for macOS)
  • Automatic screen lock after 5 minutes of inactivity
  • Install only university-approved software
  • Regular security updates must be installed within 7 days of release
  • Report lost or stolen devices immediately to IT Security

3.2 Personal Devices Used for University Work

  • Must register devices with CampusKey Device Management
  • Required to have device encryption enabled
  • Install Mobile Device Management (MDM) profile if accessing university email
  • Keep operating systems and applications updated
  • Use VPN when accessing university resources from public Wi-Fi

3.3 Network Security

  • CampusKey-Secure: Use the official university Wi-Fi network for all devices
  • VPN: Required when accessing university systems from off-campus
  • Public Wi-Fi: Avoid using unsecured public networks; use VPN if necessary
  • Firewall: Keep firewall enabled on all devices

üìß Section 4: Email and Communication Security

4.1 Phishing Prevention

Recognize and report phishing attempts:

  • Be suspicious of emails requesting passwords, payment information, or personal data
  • Verify sender email addresses (check for spoofing)
  • Hover over links before clicking to verify the destination URL
  • Never download attachments from unknown senders
  • Report suspicious emails to security@campuskey.edu
  • When in doubt, contact the sender through a verified channel

4.2 Email Best Practices

  • Never send passwords or sensitive data via email
  • Use encrypted email for sensitive information
  • Be cautious with email forwarding and sharing
  • Use official @campuskey.edu email addresses for university business
  • Enable email encryption for sensitive communications

üíæ Section 5: Data Protection

5.1 Data Classification

  • Public: Information authorized for public release
  • Internal: Information for university use only
  • Confidential: Student records, financial data, personnel information
  • Restricted: Research data, medical records, social security numbers

5.2 Data Handling Requirements

  • Store sensitive data only on approved, encrypted storage systems
  • Never store Restricted data on personal devices or cloud storage
  • Use encrypted USB drives for transporting sensitive data
  • Securely delete data when no longer needed (use university-approved deletion tools)
  • Follow FERPA guidelines for student data protection
  • Report data breaches immediately to data-protection@campuskey.edu

üö® Section 6: Security Incident Reporting

6.1 What to Report

Report the following incidents immediately:

  • Suspicious login attempts or unauthorized access
  • Lost or stolen devices with university access
  • Exposure of passwords or credentials
  • Suspected malware or virus infections
  • Data breaches or unauthorized data access
  • Phishing attempts targeting university accounts

6.2 How to Report

  • Email: security@campuskey.edu
  • Phone: (555) 123-SECU (7328) - Available 24/7
  • Online: Use the "Report an Issue" tool in CampusKey dashboard
  • In Person: IT Security Office, Library Building, Room 301

6.3 Emergency Procedures

For critical security incidents:

  • Immediately disconnect affected devices from network
  • Contact IT Security immediately
  • Do not attempt to "fix" the issue yourself
  • Preserve logs and evidence
  • Notify your supervisor or department head

7.1 Compliance Requirements

  • All users must acknowledge and comply with this security policy
  • Annual security training is mandatory for all faculty and staff
